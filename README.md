# Legit Kits Cut Guide Generator

A Python tool that converts Legit Kits quilt pattern cut guides from scanned images into formatted Excel spreadsheets and a real-time progress tracking web app.

Active quilts: **Land of the Free** (86 fabrics, 1,614 cut rows) and **Skulliver** (106 fabrics, 912 cut rows).

---

## Output

### `generate.py` → `<QuiltName>_CutGuide.xlsx`

| Sheet | Contents |
|---|---|
| **Cut Guide** | One row per piece: Fabric Code, Fabric Name, SKU, Fabric Size, Cut #, Segment ID, Sew Sequence, Page |
| **By Fabric Code** | One row per fabric with total piece count |
| **Statistics** | Summary counts and per-fabric breakdown |

Both data sheets have auto-filter enabled. The Cut Guide sheet is frozen at row 1.

### `tracking.py` → `<QuiltName>_Tracker.xlsx`

Progress tracker workbook with 6 sheets including piece counts by fabric and a block completion checklist.

Excel files are stored in `quilts/<quilt-id>/` and regenerated automatically each time the web app starts.

## Quilt Tracker Web App

A Flask web app (`quilt-tracker-app/`) for tracking quilt progress in real time, designed to run on a Raspberry Pi and be accessed from any device on the local network.

### Features

- **Multi-quilt support** — all quilts auto-discovered from `quilts/` folder; switch between them via the selector in the UI
- **8×8 block grid** with quilt overview image as background, blocks sized to match real block proportions
- Block status shown by border color: Not Started / In Progress / Complete
- Block cells show segment and piece counts (e.g. `3s 12p`)
- **Single or multi-select blocks** — plain click selects one block; Ctrl/Cmd/Shift+click adds blocks to the selection for a combined view
- **Overview panel** with progress bar and block counts when no block is selected; links to download Excel spreadsheets
- **Segments tab**: expand each segment to check off individual pieces as they are cut; segment ready checkbox unlocks only when all pieces are checked
- **Fabrics tab**: same piece data grouped by fabric color — check off pieces either here or in Segments, both stay in sync; multi-select shows merged fabric list across all selected blocks
- **Assemble tab**: assembly diagram with sewing step checklist; auto-marks block complete when all steps done
- Progress persisted across sessions in JSON files (not committed to git)

### Three-stage workflow

1. **Cut pieces** — expand a segment, check each piece checkbox as it is physically cut
2. **Segment ready** — once all pieces are checked the segment checkbox unlocks; check it when pieces are assembled into the segment
3. **Assemble block** — work through the numbered sewing steps in the Assemble tab; block is marked complete when all steps are done

### Running on Raspberry Pi

```bash
cd quilt-tracker-app
python3 app.py           # default port 3001
```

Access at `http://<pi-ip>:3001`.

**pm2 for auto-start on boot:**
```bash
pm2 start ~/legit-kits-cut-guide/quilt-tracker-app/app.py --name quilttracker --interpreter python3
pm2 save
```

**Update workflow:**
```bash
cd ~/legit-kits-cut-guide && git pull && pm2 restart quilttracker
```
Then shift-refresh in the browser for CSS/JS changes.

## Usage

### Generate cut guide Excel from existing data

```bash
python generate.py                        # all quilts
python generate.py --quilt-id skulliver   # specific quilt
```

### Validate data before generating

```bash
python lint.py
```

### Extract data from scanned images (Claude API)

```bash
python extract.py <pattern_folder>
```

The pattern folder must contain three subfolders of scanned images:

```
<pattern_folder>/
├── cut/       cut_01.jpg, cut_02.jpg, ...   (cut guide pages)
├── assy/      assy_01.jpg, assy_02.jpg, ...  (assembly guide pages)
└── overview/  overview_01.jpg, ...           (overview/kit contents pages)
```

`extract.py` calls the Claude vision API to extract structured data from each image, writes output to `quilts/<quilt-id>/`, then runs `generate.py` and `tracking.py` automatically. Each page is checkpointed so a crash only loses the page in progress; use `--resume` to continue.

Requires `ANTHROPIC_API_KEY` to be set in the environment.

## Data Format

### cut_guide_data.py

Each row in `DATA` is an 8-tuple:

```python
(fabric_code, fabric_name, sku, fabric_size, cut_num, segment_id, sew_sequence, page)
```

| Field | Type | Example | Description |
|---|---|---|---|
| `fabric_code` | str | `"AF"` | Two-letter code from the cut guide |
| `fabric_name` | str | `"Saffron"` | Color name |
| `sku` | str | `"1320"` | Legit Kits color number |
| `fabric_size` | str | `"Fat 1/8YD"` | Yardage/size included in kit |
| `cut_num` | int | `1` | Circled number on the layout diagram (Cut #) |
| `segment_id` | str | `"F3m"` | Segment identifier (Segment ID) |
| `sew_sequence` | int | `3` | Sewing sequence number for this piece |
| `page` | int | `1` | Cut guide page number |

### assembly_data.py

`BLOCKS` maps each block ID to its ordered list of fragment IDs:

```python
BLOCKS = {
    "A1": ["A1"],                              # single-fragment block
    "B7": ["B7a","B7b","B7c","B7d",            # multi-fragment block
            "B7e","B7f","B7g","B7h"],
    ...
}
```

## Data Source

Cut guides are printed booklets included with [Legit Kits](https://legitkits.com/) quilt kits. Each page of the booklet is scanned as a JPG image. Pattern data is extracted for personal organizational use.
